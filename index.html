How the aggregate score works

01 · Normalize. Every benchmark is min-max scaled to 0–100 across the models that actually report it, so a raw 95.5% on SWE-bench Verified and a raw 46.4% on HLE sit on a common footing before they're combined.
02 · Group into categories. Benchmarks roll up into five categories — Reasoning (GPQA Diamond, HLE), Coding (SWE-bench Verified, SWE-bench Pro), Math (FrontierMath T1–3, AIME), Agentic (Terminal-Bench), and Cost. A category score is the mean of the normalized benchmarks a model reports.
03 · Score cost, inverted. Cost uses a blended API price of (3×input + output) ÷ 4 dollars per million tokens, reflecting a typical 3:1 input:output mix. It's min-max normalized inverted — the cheapest model scores 100, the priciest 0. Models with no public first-party price sit out of the cost category.
04 · Three modes for cost. Performance ranks by the four capability categories only (cost ignored). Value ranks by capability composite ÷ blended price — intelligence per dollar, so cheap-but-strong models rise on merit instead of an outlier skewing the normalization. Weighted + Cost treats cost as a fifth adjustable category. This fixes the problem where a single expensive model made the cost slider swing the whole board.
05 · Rank by coverage-aware composite. The composite is the weighted mean of available category scores; weights renormalize over the categories a model has data for, so a missing benchmark lowers confidence but doesn't zero the score. The coverage column shows how many of the seven benchmarks back each model — read high-composite, low-coverage models with caution.
Read the numbers carefully. Labs report under different conditions — tools on/off, reasoning effort, agent scaffolds — so cross-lab comparisons are approximate. Scores here blend independent re-runs (Epoch AI, Scale, Artificial Analysis) with lab-claimed figures where no independent run exists. Sparse coverage can inflate a composite when the few benchmarks a model reports happen to be its strongest.
